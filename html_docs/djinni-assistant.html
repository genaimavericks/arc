Introduction

Djinni Assistant is an AI-powered conversational interface integrated into the RSW platform. It allows users to interact with data, generate insights, and visualize information using natural language queries. This documentation covers how to use Djinni Assistant effectively.

Features Overview

  • Natural Language Querying: Ask questions in plain English
  • Data Visualization: Automatically generates charts and graphs
  • Context-Aware Responses: Maintains conversation context
  • Specialized Query Categories: General, factory-specific, and custom queries
  • Query History: Tracks and saves previous interactions
  • Interactive Visualizations: Dynamic charts that respond to queries

Accessing Djinni Assistant

  1. Navigation:
  2. Click on "Djinni Assistant" in the sidebar navigation

  3. Alternatively, use the shortcut key Ctrl+D (Windows/Linux) or Cmd+D (Mac)

  4. Interface Components:

  5. Chat Area: Main conversation interface
  6. Query Input: Text field at the bottom
  7. Visualization Panel: Displays charts and graphs
  8. Query Categories: Sidebar with query suggestions
  9. Settings: Configuration options

Using Djinni Assistant

Basic Interaction

  1. Starting a Conversation:
  2. Type your question in the input field
  3. Press Enter or click the Send button

  4. Wait for Djinni to process and respond

  5. Query Types:

  6. Informational: "What is the total production for June?"
  7. Comparative: "Compare machine utilization percentage among all factories"
  8. Analytical: "What's causing the increase in downtime?"

  9. Predictive: "What will be the revenue for factory 3 over the next 3 months?"

  10. Refining Queries:

  11. Ask follow-up questions to drill down
  12. Specify time periods: "Show me data from last quarter"
  13. Filter by categories: "Only show Factory 2 data"

Visualization Features

  1. Chart Types:
  2. Bar charts for comparisons
  3. Line charts for trends
  4. Pie charts for proportions
  5. Radar charts for multi-dimensional metrics
  6. Tables for detailed data

  1. Interacting with Visualizations:
  2. Hover over data points for details
  3. Click on legend items to toggle visibility
  4. Use zoom controls for detailed views

  5. Download charts as images or data

  6. Customizing Visualizations:

  7. Request specific chart types: "Show as a bar chart"
  8. Adjust time ranges: "Show monthly instead of quarterly"
  9. Change metrics: "Show by revenue instead of units"

Factory-Specific Queries

Djinni Assistant provides specialized queries for factory analytics:

  1. Production Queries:
  2. "What is the production volume for Factory 2?"
  3. "Compare production volumes across all factories"

  4. "Show production trend for the past year"

  5. Revenue Queries:

  6. "What will be the revenue for Factory 3 over the next 3 months?"
  7. "Compare profit margins across factories"

  8. "Show revenue breakdown by product line"

  9. Efficiency Queries:

  10. "Compare machine utilization percentage among all factories"
  11. "What's the downtime percentage for Factory 1?"

  12. "Which factory has the highest efficiency?"

  13. Quality Metrics:

  14. "Show quality metrics for Factory 4"
  15. "Compare defect rates across production lines"
  16. "What's the trend in quality scores?"

Query History and Management

  1. Viewing History:
  2. Click the "History" tab to view past conversations
  3. Search through previous queries

  4. Filter by date or query type

  5. Saving Important Queries:

  6. Star important conversations for quick access
  7. Create named bookmarks for frequent queries

  8. Export conversation logs for reporting

  9. Managing Context:

  10. Start new conversation with the "New Chat" button
  11. Clear context with "Reset Conversation"
  12. Reference previous queries: "Like the last query but for Factory 2"

Advanced Features

  1. Data Integration:
  2. Query across multiple data sources
  3. Combine DataPuur and KGInsights data
  4. Access historical and real-time information

Best Practices

  1. Effective Querying:
  2. Be specific about metrics and time periods
  3. Start with broad questions, then refine

  4. Use consistent terminology

  5. Optimizing Results:

  6. Provide feedback on helpful/unhelpful responses
  7. Rephrase questions if results aren't as expected

  8. Use suggested queries as templates

  9. Collaboration[Planned for Future Releases]:

  10. Share insights with team members
  11. Use in meetings for dynamic data exploration
  12. Create standard query templates for teams

Troubleshooting

  1. Query Not Understood:
  2. Rephrase using simpler terms
  3. Check for typos or ambiguous language

  4. Use suggested query templates

  5. Incorrect Results:

  6. Verify data sources are up to date
  7. Check for filters that might be limiting results

  8. Specify time ranges explicitly

  9. Performance Issues:

  10. Limit complex queries during peak usage times
  11. Break complex questions into simpler ones
  12. Refresh the browser if response times are slow

Example Query Workflow

Here's an example of how to use Djinni Assistant effectively:

  1. Start with a general question: "Show me machine utilization across all factories"

  2. Djinni displays a pie chart showing utilization percentages.

  3. Ask a follow-up question: "Which factory has the lowest utilization?"

  4. Djinni identifies Factory 3 with 56.23% utilization.

  5. Drill down further: "What are the main causes of downtime in Factory 3?"

  6. Djinni provides a breakdown of downtime causes.
