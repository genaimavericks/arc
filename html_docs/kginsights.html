Introduction

KGInsights (K-Graff) is the knowledge graph management module of the RSW platform. It enables users to create, visualize, explore, and derive insights from complex relationship networks within their data. By representing data as interconnected entities and relationships, K-Graff helps uncover hidden patterns and connections that might be missed in traditional tabular analysis.

Features Overview

  • Graph Generation: Create knowledge graphs from structured data
  • Graph Visualization: Interactive visual exploration of relationships
  • Graph Management: Edit, update, and maintain knowledge graphs
  • Schema Management: Define and manage entity and relationship types
  • Network Analysis: Calculate centrality, clustering, and other metrics
  • Path Discovery: Find connections between entities
  • Pattern Recognition: Identify recurring structures and motifs

Generate Graph

Troubleshooting

Graph Generation Issues

  1. Processing Failures:
  2. Check data source validity
  3. Verify entity identifier uniqueness
  4. Review relationship mappings

  5. Check for memory limitations

  6. Schema Conflicts:

  7. Ensure data conforms to schema
  8. Check for missing required properties

  9. Verify relationship constraints

  10. Performance Problems:

  11. Consider sampling for large datasets
  12. Break complex graphs into subgraphs
  13. Optimize entity and relationship selection

Visualization Issues

  1. Rendering Problems:
  2. Reduce graph size for better performance
  3. Use filtering to focus on subgraphs
  4. Try alternative layout algorithms

  5. Update browser or clear cache

  6. Layout Issues:

  7. Adjust layout parameters
  8. Pin important nodes
  9. Use hierarchical layouts for complex graphs
  10. Apply clustering to simplify visualization

Analysis Issues

  1. Slow Analysis Computation:
  2. Limit analysis to relevant subgraphs
  3. Use approximation algorithms for large graphs

  4. Schedule complex analyses for off-peak times

  5. Unexpected Results:

  6. Verify algorithm parameters
  7. Check for data anomalies
  8. Consider alternative metrics
  9. Consult domain experts for interpretation
